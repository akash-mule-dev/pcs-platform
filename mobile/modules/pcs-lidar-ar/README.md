# pcs-lidar-ar — native iPad LiDAR mixed-reality engine

A local Expo native module (iOS only) that renders the AR model with Apple
**RealityKit** so it can use the iPad's **LiDAR** sensor — something the Viro
engine physically cannot do. It backs the new LiDAR modes in the AR view
(`ARExperienceRK`), shown only on **iPad + LiDAR**; every other device keeps the
existing Viro path untouched.

> ⚠️ Device-only. ARKit/RealityKit do not run in the iOS Simulator. Build to a
> physical iPad Pro (or LiDAR iPad). `mobile/ios` is a gitignored Expo prebuild —
> always regenerate it; never hand-edit Xcode (changes are wiped by prebuild).

## The LiDAR modes (compare these on the iPad)

| Mode | Uses | What you should see |
|---|---|---|
| **Mesh Scan** | `sceneReconstruction` + `showSceneUnderstanding` | The LiDAR mesh drawn over the room — proof the sensor is live |
| **Occlusion** | scene-depth `.occlusion` + `.personSegmentationWithDepth` | Real beams/walls **and your hand** correctly hide the model |
| **Anchored** | LiDAR raycast + plane anchor | Rock-steady lock to the real surface (stability-first) |
| **Physics** | `.physics`/`.collision` on the mesh | The model rests on / collides with the real floor |

Modes switch **live** with no camera flicker (they flip RealityKit's
`environment.sceneUnderstanding.options` / `debugOptions`). Only entering/leaving
the hand-occlusion part of Occlusion re-runs the ARKit session — without reset
flags, so the placed model and tracking survive.

## Build & run on the iPad

From `mobile/`:

```bash
# 1. Regenerate the native project (re-runs Expo autolinking; this module is
#    auto-discovered from modules/ — no Podfile/package.json edit needed).
npx expo prebuild --clean -p ios

# 2. Install pods.
cd ios && pod install && cd ..

# 3. Build + run on the connected iPad (set the Signing Team for com.fabrixr.pcs
#    in Xcode once if auto-signing fails: open ios/PCS.xcworkspace).
npx expo run:ios --device

# 4. Start Metro for the dev client.
npx expo start --dev-client
```

After ANY native edit you must rebuild (steps 1–3) — Fast Refresh only reloads JS.

### Step 0 (first build): no model loading yet — and that's intentional

Out of the box this module has **no third-party dependency** and renders a
**placeholder box** instead of your GLB. That's deliberate: it lets you validate
every LiDAR mode (occlusion, hand-occlusion, stability, physics) on the iPad in
one build, before dealing with model loading. Flip the **🛰 LiDAR** engine toggle
in the AR view, move around, pass your hand in front in Occlusion mode.

### Step 1: load your real GLB models (add GLTFKit2)

RealityKit can't load GLB natively, so to render your actual models add
**GLTFKit2** (github.com/warrenm/GLTFKit2, 0.5.15). The Swift is already written
behind `#if canImport(GLTFKit2)` — once the package is linked it activates
automatically. It is **not** on CocoaPods (no pod), so use one of:

- **Option A — SPM (cleanest):** add the package to the app target. Because
  `ios/` is regenerated by prebuild, do it via a config plugin (mirror
  `plugins/withViroMainApplicationFix.js`) using `withXcodeProject` to inject the
  SPM package `https://github.com/warrenm/GLTFKit2` (from `0.5.15`, product
  `GLTFKit2`) on every prebuild, then register the plugin in `app.config.js`.
- **Option B — vendored xcframework (most self-contained):** download
  `GLTFKit2.xcframework.zip` from the 0.5.15 GitHub release, unzip it to
  `modules/pcs-lidar-ar/ios/GLTFKit2.xcframework`, and **uncomment**
  `s.vendored_frameworks = 'GLTFKit2.xcframework'` in `PcsLidarAr.podspec`. Commit
  the framework (this `modules/` dir is source-controlled). No plugin needed.

Then rebuild (steps 1–3). The native loader (`GltfModelLoader.swift`) takes the
same `file://` GLB `useRemoteModel` already caches — no server-side conversion.

## Known first-build iteration points (device-only verification)

These were flagged during research; expect to confirm/tune on the iPad:

1. **Shading of IFC GLBs.** IFC-derived GLBs carry no vertex normals; GLTFKit2
   doesn't synthesize them, so the loader assigns a flat **UnlitMaterial** (looks
   right without normals). If you want lit/PBR shading, generate normals — but
   confirm unlit is insufficient first.
2. **GLTFKit2 linking.** No CocoaPod exists — never `s.dependency 'GLTFKit2'`
   (pod install would fail). Use Option A or B above.
3. **Engine-toggle placement.** The 🛰 LiDAR toggle is pinned bottom-center over
   the Viro view (`ARViewScreen` `styles.engineSwitch`); nudge `bottom` if it
   overlaps an open tool panel.
4. **Large models.** Big IFC GLBs parse synchronously inside the loader — test the
   largest expected model on the lowest iPad; the loading pill is already shown.
5. **Physics mode** is the stretch mode; if collision-from-mesh misbehaves, drop
   it from `REALITYKIT_MODES` in `ar/types.ts` and ship Mesh/Occlusion/Anchored.

To disable the whole engine instantly: set `REALITYKIT_ENGINE_ENABLED = false` in
`src/screens/model-viewer/ARViewScreen.tsx` — the toggle vanishes and Viro is the
sole path again.
